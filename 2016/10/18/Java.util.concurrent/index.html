初识 Java.util.concurrent

concurrent是一个在并发编程中的工具类包，学习之后简单记录下，内容都是API或者网络上博客的一些整理，我尽量按照自己的理解描述的清晰一点，内容比较浅，初识嘛，问题不大~

Excutors

主要方法：

  • newCachedThreadPool: 可重用的固定线程数量的线程池
  • newFixedThreadPool: 无界的按需创建的线程池，60s自动回收未被使用的线程。
  • newSingleThreadExcutor: 单个线程的Executor

可以看到包含了5个线程的线程池只能执行5个线程，只有前5个线程执行完毕，才能继续执行。

多说一句，这些提交的线程被存放在一个无界队列里，这个无界队列是一个有序队列。（BlockingQueue）

ThreadPoolExcutors

当被问到怎么确定线程池的大小的时候，我发觉对很多知识还是欠缺，这边补充一下
线程池的相关信息。

ThreadPoolExcutors是线程池中最核心的一个类，先看下构造方法：

前三个构造方法都是调用第四个构造方法进行实例化的。

  • corePoolSize: 核心池的大小。当创建完毕线程池的的时候，并不会创建线程，只有当用户进行请求的时候才会创建线程。除非调用prestartAllCoreThreads()或者prestartCoreThread()方法。当创建完线程池的时候，默认线程为0.当有请求的时候创建线程直到corePoolSize，如果还有线程继续进来那么就会放到缓存队列里。
  • maximumPoolSize：线程池最大线程数，它表示在线程池中最多能创建多少个线程.
  • keepAliveTime:表示过了多久线程会终止，一般情况下只有在线程数目大雨corePoolSize的时候才会起作用。到达最大空闲时间的线程会停止。但是如果调用了allowCoreThreadTimeOut(boolean)方法，在线程池中的线程数不大于corePoolSize时，keepAliveTime参数也会起作用，直到线程池中的线程数为0；
  • unit:keepAliveTime的时间单位，有各种格式。
  • workQueue:一个阻塞队列，用来存储要执行的任务，会对线程池产生重大影响。

Java提供七个阻塞队列，这边先提一下，有机会会详细说一下这些阻塞队列。
ArrayBlockingQueue ：一个由数组结构组成的有界阻塞队列。
LinkedBlockingQueue ：一个由链表结构组成的有界阻塞队列（常用）。
PriorityBlockingQueue ：一个支持优先级排序的无界阻塞队列。
DelayQueue：一个使用优先级队列实现的无界阻塞队列。
SynchronousQueue：一个不存储元素的阻塞队列（常用）。
LinkedTransferQueue：一个由链表结构组成的无界阻塞队列。
LinkedBlockingDeque：一个由链表结构组成的双向阻塞队列。

  • threadFactory: 线程工厂，用来创建线程。
  • handler:表示当拒绝处理任务时的策略.

ThreadPoolExecutor有几个很重要的方法：

  • execute():这方法实际上是Exexutor中声明的方法，是ThreadPoolExecutor的核心方法，这个方法可以向线程池提交一个任务，交由线程池去执行。
  • submit():其中调用了execute方法，不同的地方在于他返回了Future.
  • shutdown():关闭线程池。
  • shutdownNow()：同样关闭线程池，不过是now.

线程池的状态

我们看看代码是咋写的：

* The runState provides the main lifecycle control, taking on values:
*
*   RUNNING:  Accept new tasks and process queued tasks
*   SHUTDOWN: Don't accept new tasks, but process queued tasks
*   STOP:     Don't accept new tasks, don't process queued tasks,
*             and interrupt in-progress tasks
*   TIDYING:  All tasks have terminated, workerCount is zero,
*             the thread transitioning to state TIDYING
*             will run the terminated() hook method
*   TERMINATED: terminated() has completed
*
* The numerical order among these values matters, to allow
* ordered comparisons. The runState monotonically increases over
* time, but need not hit each state. The transitions are:
*
* RUNNING -> SHUTDOWN
*    On invocation of shutdown(), perhaps implicitly in finalize()
* (RUNNING or SHUTDOWN) -> STOP
*    On invocation of shutdownNow()
* SHUTDOWN -> TIDYING
*    When both queue and pool are empty
* STOP -> TIDYING
*    When pool is empty
* TIDYING -> TERMINATED
*    When the terminated() hook method has completed
*

写的挺明白。

这边为什么要采取左移来实现，有什么好处呢? 目前还不知道….机会合适的时候可以研究一下….

好，到现在我们尝试着自己理解一下execute()的实现原理是什么：

　不过在java doc中，并不提倡我们直接使用ThreadPoolExecutor，而是使用Executors类中提供的几个静态方法来创建线程池：

newFixedThreadPool创建的线程池corePoolSize和maximumPoolSize值是相等的，它使用的LinkedBlockingQueue；

　　newSingleThreadExecutor将corePoolSize和maximumPoolSize都设置为1，也使用的LinkedBlockingQueue；

　　newCachedThreadPool将corePoolSize设置为0，将maximumPoolSize设置为Integer.MAX_VALUE，使用的SynchronousQueue，也就是说来了任务就创建线程运行，当线程空闲超过60秒，就销毁线程。

　　实际中，如果Executors提供的三个静态方法能满足要求，就尽量使用它提供的三个方法，因为自己去手动配置ThreadPoolExecutor的参数有点麻烦，要根据实际任务的类型和数量来进行配置。

　　另外，如果ThreadPoolExecutor达不到要求，可以自己继承ThreadPoolExecutor类进行重写。

Semaphore

维护一组信号量，对号码进行计数，就像去医院看病，1个医生维护一个信号量，1个房间2个医生，就维护2信号量，出1个，进1个，就这样。

ReentrantLock

一个可重入的互斥锁。 可重入就代表某一个线程可以多次获得一个锁。可以理解为加强版的synchronized。

先看API上的例子

一个线程申请锁，如果这个锁没有被其他线程占用呢，那么就立即返回。如果这个锁已经被当前线程占用了呢，那么就将保持计数加1，如果被别的进程占用了的话，大部分情况下这个锁就处于休眠状态，然后等待获取锁，锁的个数可以通过 getHoldCount()方法来查看。

他和synchronize的区别在于，他的功能多一些，但是synchronize优化做的更好一些，效率较高。

简单的应用就是加个锁，没啥可说的。

ConcurrentHashMap

再说ConcurrentHashMap前，我们先看看HashMap和HashTable.

HashMap不是线程安全的，put方法会导致死循环，你一跑，会发现CPU利用率达到100%。HashTable是线程安全的，但是效率太低。HashTable效率低下的原因是多个线程会争抢一把锁。那如果一个容器可以拥有多把锁，每个锁只锁一部分数据呢？这就是ConcurrentHashMap.
