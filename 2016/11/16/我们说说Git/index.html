简单介绍

关于Git这块大家肯定不陌生，因为目前的工作没有在用Git，我本身也就是自己了解下。
（其实是因为最近工作比较累，想更新点好写，轻松的~）
行吧，咱们开始了~

完全的教程呢，应该是从发展历史开始一直到balabala结束，不过呢我这就是简单的一随笔，所以呢，我就写点常用的，细节什么的，再补充。

Git简单来说就是分布式的版本控制系统，就是大家可以一起提交版本，使teamwork变得更惬意的一工具，随之而来的还有世界上最大的同性交友网站Github。Linus Torvalds就是吊。

##Git基础

特点：

  • 记录快照，并非差异
  • 大多数操作本地执行，不需联网

###文件的三种状态
在Git中， 所有的文件都只有这三种状态：已提交(committed)，已暂存(staged)，已更改(modified)。

已提交代表文件已经保存到本地的数据库这中了。
已暂存表示已经把修改的文件添加到了下次提交保存的名单中。
已更改表示已经修改了某文件，但是还没有提交保存。

每个项目都有一个Git目录，他是用你来保存对象元数据和对象数据库的地方，很重要。每次克隆镜像仓库的时候，实际上就是copy这个目录。而所谓的暂存区只不过是一个简单的文件，可以简单理解为索引文件。

Git使用

###安装git
我使用的是Mac,建议先安装Homebrew，然后安装Git。具体的简单查一下就行。

###常用的一些操作

mkdir adultswim
cd adultswim
git init

然后我们搞了个文件， 想把它push上去，比如说我们加了个readme.txt

我们要用 git add readme.txt 这样就是告诉git，我们加了这么个东西。加完了该提交了，这么弄：git commit -m “Just add readme.txt”

同样可以使用git status来查看文件修改的状态。但是如果我要是想看我到底改了些什么，那该怎么办呢？

使用git diff就可以了。

我们提交了很多版本，如果我们想回退到某一个版本，先用git log看下我们的提交日志，我们可以看到最新的三个日志。

你会看到大约这么一串东西:

$ git log

Author: Michael Liao askxuefeng@gmail.com
Date: Tue Aug 20 15:11:49 2013 +0800

append GPL

commit cb926e7ea50ad11b8f9e909c05226233bf755030

这里的commit后面有一个串，这个串就是你的ID,下面就是你git commit -m “里面的内容”， 你就知道你想回滚到哪个版本了。

在git中, HEAD表示当前版本，上一个版本是HEAD^,再上一个版本就是HEAD^^.

所以我们就可以使用：git reset –hard HEAD^,我们就到了”add distributed”.
再看下 git log,就剩下俩版本了

如果我们又想回去该怎么办？
使用 git reset –hard 3628164fb26d48395 就行了。 我要是不记得3628…那一串怎么办？窗口已经被我关掉了。那没问题，有个命令叫做： git reflog

ea34578 HEAD@{0}: reset: moving to HEAD^
3628164 HEAD@{1}: commit: append GPL
ea34578 HEAD@{2}: commit: add distributed
cb926e7 HEAD@{3}: commit (initial): wrote a readme file

行了，这下找到了。

多说一句，结合前面说的暂存区和工作区。你用git add, 就把文件加到了暂存区。然后git commit,你就提交到了工作区。你工作的地方就叫工作区。而暂存区，就存放在你的.git目录中。

如果你在工作区添加了一些东西，但是没有commit，这时候使用 git checkout -- file,如果你commit了，那么先回退到上一个版本git reset HEAD file,然后再git checkout -- file

创建远程仓库：

我也记不住，到时候查一下就行,后面网址是你github上面那个地址,比如我这个博客的github是https://github.com/felisnigreps/felisnigreps.github.com
$ git remote add origin git@https://github.com/felisnigreps/felisnigreps.github.com.git

把本地内容推送到仓库
git push -u origin master
只需要第一次加上-u,以后就不用了

克隆远程仓库：
git clone git@XX

分支管理：

分支这块呢，github主要用的指针，远离比较好理解，不过需要上网查一下，我这就简单写点命令。

比如我要创建个dev的分支：git checkout -b dev

加上-b代表着创建并且切换，相当于下面两条命令：git branch dev,git checkout dec

如果你想要查看分支呢，就用git branch,会显示出所有分支，当前分支前面会有个*号。 然后我们就按照正常来就行了，git add, git commit balabala…

然后我们需要切回到master,使用git checkout master就行了。Git做这些东西非常快，因为切换的都是指针。

回去之后只能看到master看到的修改，行了，我们现在要把他合并，这就要使用git merge dev. 行了，合并完了，现在可以删除dev分支了,git brance -d dev, 因为创建，合并，删除分支的速度非常快，所以在一般情况下，建议大家建立新的分支来进行更改，改完了再合并，毕竟这样更安全嘛。

Bug分支

我们现在要改个bug，现在的分支还没写完，咋办？git status看下，显示还有东西没提交，保存起来用 git stash，在用git status瞅一眼，行了。完活。然后创建分支该呗，git checkout master git checkout -b bug01,然后修改，修复完了add, commit, 然后合并分支. git checkout master git merge --no-off -m "fix bug01" bug01,然后删除分支。git branch -d bug01(–no-ff参数，表示禁用Fast forward,fast forward模式删除分支后，会默认删除分支信息)。行了，现在可以回去继续工作了，git checkout dev git status发现dev的东西没有了，保存的东西去哪了？ git stash list 就行了，然后恢复git stash apply stash@{0}

我们来梳理一下工作流程。 你先从github colne出最新版本，然后创建自己的分支bug/feature, 做好你的更改，然后合并分支到dev.
