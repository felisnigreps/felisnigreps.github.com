Java IO & NIO

刚开始看IO的时候还没有这个blog,昨天用到感觉没有以前那么熟练了。在公司想找以前联系时候的源码还没找到，不过问题不大，今天过一遍复习一下。

以上是一些处理IO的类，我们根据情况来选择不同的方式来处理问题。

文件

读文件

FileReader和FileInputStream

取文件

FileWriter和FileOutputStream

随机存取文件

使用RandomAccessFile对文件进行存取，这个存取并不是‘随机’，而是可以让你跳过某些字符来进行读写。

文件和目录信息的获取

通过File类来获取文件和目录的信息。

管道

管道可以为JVM中两个线程进行通讯。管道是PipedInputStream和PipedOutputStream. 一个PIS流应该和一个POS流相关联。一个通过PIS流写入的数据可以被另一个和其相关联的POS流读取出来。

尝试着用try-with-resource来关闭流，结果未遂。各种错误都有。上网搜索了一圈也未果。

字节流一般用*Stream来命名，字符流一般用Reader/Writer来命名。

RandomAccessFile

RAF允许你来回读写文件，也可以替换文件中的某些部分。

BufferedInputStream

BIS这是个好东西，他会设置一个缓冲区进行读取。快，读取大量数据的时候建议用他。

DataInputStream

可以按照类型读取，看看API上的方法一下就都知道了

NIO

IO一看也没什么太细节的东西，都是读了就了解了，咱们说说NIO.

Java NIO有三个核心部分

  • Selector
  • Channel
  • Buffer

Channel and Buffer

C和B之间和可传递数据，他们都有不同的实现

Channel

  • FileChannel
  • DataGramChannel
  • SocketChannel
  • ServerSocketChannel

Buffer
就是一些数据类型的buffer

  • ByteBuffer
  • ShortBuffer
  • DoubleBuffer
  • LongBuffer
  • IntBuffer
  • CharBuffer
  • FloatBuffer

Selector
Selector可以在单线程下管理多个channel，要使用selector,你需要向Selector注册Channel,然后调用他的seek()，这个方法会一直阻塞到他注册的通道有事件就绪，然后返回，这样线程就会处理这个事件。

more about Channel

既可以从Channel读数据，又可以谢数据到Channel.Channel可以进行异步的读写。Channel中的数据总是要写入一个buffer中，或者从一个buffer中读，就像上面图示的那样。

我们刚说了有四种channel

FileChanner: 主要负责从文件中读取数据,是阻塞IO。
DataGramChannel: 通过UDP读写数据
SocketChannel: 通过TCP读写数据
ServerSocketChannel: 监听新的TCP连接，每有一个连接就注册一个SocketChannel

刚看buffer的代码有一点蒙，缓了那么一下，才想起来他的工作机制，buffer有几个属性，我们需要了解一下：

  • capcity: buffer的容量大小，一般用buffer的子类的allocate(num)方法来定义大小。
  • limit: buffer可用的大小。
  • position: 下一个需要读取元素的索引。
  • mark: 一个记录当前position的值，当使用mark()的时候会设置成mark=position,调用reset方法可以使position返回到mark的位置。

还有一些常用的方法：

  • flip:从写模式切换到读模式，position变为0，limit变为position
  • mark:设定标记点，和reset一起用
  • rewind:重读buffer中的数据，position变为0，limit保持不变
  • clear:清空buff，但是数据还在。position变为0，limit变为captity

Selector

前面我们提过，Selector是用来管理多个channel的，那么我们使用Selector的目的是什么呢？答案就是可以通过更少的线程来管理更多的channel.

Selector只能管理非阻塞Channel,我们上面说过Channel一般有四个，FileChannel是阻塞的，ServerSocketChannel是管理SocketeChannel.就剩下DataGrapChannel和SocketChannel了，ok，咱们稍微写一写，写到哪了就顺带着介绍下

SocketChannel

SocketChannel是一个连接到网络上的TCP套接字的通道。

读写到buffer和其他channel一样。

ServerSocketChannel

SSC是一个可以监听新TCP连接的通道。

DataGramChannel

这个是监控UDP的，操作几乎和SocketChannel一致。

总结

IO面向流，NIO面向缓冲。

IO阻塞，NIO非阻塞。
