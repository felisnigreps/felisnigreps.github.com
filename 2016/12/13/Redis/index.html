Redis

let’s get it started

Redis是一种持久化的存储器内存储(in-memory persistent store)

先输入这么一条命令：

set users:Feli {name:Feli,planet:moon,like:[talk]}
get users:Feli

对于Redis来讲，关键字就是一切，而值没有任何意义，你不太可能通过值来查哪些用户在moon上。这对我们来说感觉有些不完善，但是Redis并不是一种通用方案，他只是为了某种特定的场景而生。

setbit 和 getbit

典型的命令如下：

setbit mykey 66666 1
getbit mykey

刚开始的时候不理解，这到底是什么？可以这么理解，有一个2的32次幂长的bit数组，每个数组上写的是0 or

1.通过这个命令，你可以给这个数组的某个位置设置一个值。怎么使用，存取异常之快，怎么用，发挥你的想象力吧。

Redis是单线程执行的，所以会保持原子性。

5大数据类型：

  • String
  • Set(列表):String类型的无序集合,不可重复,maxLengh为pow(2,32),是通过hashtable来实现的所以删除查找复杂度都为O(1).
  • list(集合)：其实就是每个元素都是String的双向链表,maxLengh为pow(2,32)
  • sorted set(有序列表):在set基础上增加了一个顺序属性（double score），这个属性在添加元素的时候可以指定。添加了原色zset会重新排序。
  • hashes(哈希表): String类型的field和value的映射表，添加删除的复杂度O(1)

Redis支持乐观锁，通过版本号来控制。版本号不符合则取消回滚。（提交版本必须大于记录版本)

但是如果一个事务中某一部分出错了，他只回滚错误的那部分。而执行正确的不会回滚。

持久化

Redis是一个支持持久化的数据库，有两种方式可以支持Redis持久化。
Snapshotting:默认方式。简单说下原理。可以设置每隔多少秒有多少次修改就进行持久化。
save 900 1 #900s内有一次改变，下方同理。
save 300 10
save 60 10000

当开始持久化的时候redis调用fork，创建子线程。然后使用CoW的方式进行持久化，会把二进制文件保存在.dump文件中。如你所见，RDB如果意外崩溃的话，那么你会丢失一小部分的数据，如果你想避免这种情况，我们来看第二种持久化方式。

AOF:Redis在接收到命令的时候会把每一个写命令追加到文件中(默认appendonly.aof)，然后启动的时候再重新写入。当然，这样的问题就是会有大量的命令。比如我们使用了incr test 10000次，那么会有10000条命令。直接就相当于 incrby test 100000.

Redis提供了bgrewriteaof命令。
