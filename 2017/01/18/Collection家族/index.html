Collection家族

这是Java基础中不可缺少的部分，以前看过很多次，但总是记不牢，这次在这边简单总结下。方便复习。

Collection也包含了基本的数据结构，这边也会在介绍的时候简单同提一下。

很多图片和内容都是抄自于网上的博客，侵删。

集合体系

Collection

Set:一种不包含重复元素Collection,他维持自己内部的顺序，所以随机访问对他没有意义（无序）。可以存在null值但是只能存在一个。

  • HashSet: 号称查询速度最快的集合，其内部是HashCode实现的，他不保证set的顺序。也不保证顺序的不变。
    不允许出现重复元素；
    不保证集合中元素的顺序。哈希算法来的~
    允许包含值为null的元素，但最多只能有一个null元素。

  • LinkedHashSet: 他介于HashSet和TreeSet之间，他是一个Hash表，同时也维护了一个双链表来记录顺序。

  • TreeSet: 是一个总处于排序装的set,内部以TreeMap作为实现（使用红黑树算法）。元素按照顺序排列，但是add()/remove()等方法的复杂度为O(logn).

  • TreeSet
    不允许出现重复元素；
    集合中元素的顺序按某种规则进行排序
    不允许包含值为null的元素

List:是一个接口

代表有序的Collection,他按照某种特定的插入顺序来维护排列顺序。可以根据位置精确的插入数据，并且可以根据索引访问元素,以下是他的实现。

  • ArrayList：一个动态数组，是我们最常用的集合。它可以插入null,重复的数据。每个ArrayList都有一额默认大小（10），每次插入元素的时候都会进行容量检查。当快要达到容量的时候就会进行扩容，如果知道需要的ArrayList的大小的话，最好在初始化的时候就声明他的大小来节省资源。
    ArrayList擅长随机访问，ArrayList是非同步的。

  • Vector： 和Array几乎一致，不过区别在于他是线程安全的。但是效率很低，不建议使用。

  • LinkedList：它是一个双向链表，可以更好的进行插入和删除，除了ArrayList提供的方法之外，他还有get/remove等方法获取头部的元素。它查找的方式和ArrayList有所不同，他不支持随机查找，每次查找都需要从头/尾进行遍历来查找元素。

Map

Map接口，没有继承Collection接口，它是独立的一个接口。它使用key-value的键值对存储数据。子类有HashMap，TreeMap，LinkedHashMap等。

HashMap是无序的，TreeMap和LinkedHashMap是有序的，TreeMap实现了SortMap接口，默认根据key的升序进行排序。而LinkedHashMap是保存的插入的顺序。

HashMap

什么会影响HashMap的性能?

首先我们要知道有两点,初始容量和加载因子. 首先容量要是2的次幂,为什么这么做?因为可以减少碰撞,这和根据key算hashcode的算法有关,详情可以参见源码. 第二点就是容量,比如说，我们有1000个元素new HashMap(1000), 但是理论上来讲new HashMap(1024)更合适，不过上面已经说过，即使是1000，hashmap也自动会将其设置为1024。 但是new HashMap(1024)还不是更合适的，因为0.751000 < 1000, 也就是说为了让0.75 size > 1000, 我们必须这样new HashMap(2048)才最合适，既考虑了&的问题，也避免了resize的问题。

HashMap的迭代器(Iterator)是fail-fast迭代器,Hashtable的enumerator迭代器不是fail-fast的.所以当其他线程改变了HashMap的结构,将会抛出ConcurrentModificationException.
当一个或多个线程正在遍历一个集合Collection，此时另一个线程修改了这个集合的内容（添加，删除或者修改）。这就是并发修改

工具类

Iterator

所有的集合类，都实现了Iterator接口，这是一个用于遍历集合中元素的接口，主要包含以下三种方法：
1.hasNext()是否还有下一个元素。
2.next()返回下一个元素。
3.remove()删除当前元素。
